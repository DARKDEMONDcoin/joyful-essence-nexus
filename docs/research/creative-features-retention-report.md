## 1. ChatGPT (OpenAI) — كل النماذج والميزات الإبداعية

### 1.1 Canvas
- **ما هو**: واجهة جانبية (side-pane) منفصلة عن الشات، تفتح تلقائيًا عند طلب كتابة نص طويل أو كود. تسمح بالتعاون المباشر: تحديد جزء من النص/الكود وطلب تعديل محدد ("اجعل هذا أقصر"، "أضف تعليقات")، مع أزرار سريعة (Suggest edits, Adjust reading level, Add emojis, Fix bugs, Add logs, Port to another language).
- **آلية العمل تقنيًا (UX pattern)**: split-view state manager يحتفظ بنسخة "canvas document" منفصلة عن سجل المحادثة، مع **diff view** يُظهر التغييرات (مثل git diff) بدل استبدال النص كاملاً؛ history/undo داخل الـ Canvas نفسه؛ يدعم تشغيل كود Python مباشرة (preview iframe) وعرض HTML/React حي.
- **لماذا يحبها المستخدمون**: تفصل "التفكير" عن "المُنتَج النهائي"، تقلل إعادة اللصق الكامل، تعطي إحساس التحكم المباشر (direct manipulation) بدل الحوار النصي الخطي فقط.
- المصدر: openai.com/index/introducing-canvas, help.openai.com/articles/9930697

### 1.2 Tasks (Scheduled Tasks)
- جدولة ChatGPT لتنفيذ مهمة متكررة أو لمرة واحدة (تذكير، تلخيص أخبار يومي، متابعة سعر سهم) وإرسال **push/email notification** عند التنفيذ.
- UX: عبر واجهة شبيهة بـ "cron جملة طبيعية" (كل يوم اثنين الساعة 9، كل صباح...)، مع صفحة إدارة مركزية للمهام النشطة.
- سبب الجاذبية: يحول ChatGPT من أداة "رد فعل" (reactive) إلى أداة "استباقية" (proactive) تعاود الظهور للمستخدم بدون طلب منه — وهذا فعليًا **إعادة تفعيل** (re-engagement) مبرمجة.

### 1.3 Memory
- نوعان: (أ) **Saved memories** الصريحة (يذكر المستخدم شيء فيحفظه الموديل بشكل دائم عبر المحادثات)، (ب) **Chat history reference** الضمنية (يستخلص سياق من محادثات سابقة تلقائيًا دون طلب صريح).
- UX: صفحة إعدادات لعرض/حذف/تعديل الذكريات فرديًا (transparency & control)، toggle للتفعيل/التعطيل الكامل.
- الأثر النفسي: **تخصيص عميق (deep personalization)** يخلق شعور "يعرفني" — ركيزة أساسية لبناء رابطة استمرارية شبه علائقية.

### 1.4 Custom GPTs & GPT Store
- بناء "شخصية/أداة" مخصصة بدون كود (system prompt + knowledge files + actions API + capabilities toggles)، نشرها في متجر عام مع تصنيفات وتقييمات ("Trending", "Top Picks").
- آلية: كل GPT له namespace منفصل من المحادثات + ذاكرة اختيارية خاصة به.
- الجاذبية: **social proof** (عدد الاستخدامات، الترتيب) + امتلاك (ownership) يشجع الصناع على الترويج، وهو حلقة نمو (growth loop) شبيهة بمتاجر التطبيقات.

### 1.5 Voice (Advanced Voice Mode)
- محادثة صوتية زمن-حقيقي (real-time, low latency ~300ms) مع كشف المقاطعة (interruption handling)، نبرة عاطفية، ورؤية عبر الكاميرا (يرى الشاشة/العالم أثناء الحديث في تطبيق الموبايل).
- UX pattern: دائرة/كرة متحركة (visualizer) تنبض مع الصوت — إشارة بصرية لـ"الاستماع الفعلي" تبني ثقة الحضور (presence).

### 1.6 Reasoning models (o1 / o3 / GPT-5 thinking) — Thinking Traces UI
- عرض "سلسلة تفكير مختصرة" (summarized chain-of-thought) أثناء المعالجة، بعناوين متغيرة حية ("قارن المصادر…"، "يتحقق من المنطق…") بدل نص خام كامل (لأسباب أمان/تنافسية).
- زر "Show thinking" قابل للطي/التوسيع.
- الأثر: يحول زمن الانتظار (latency) من "فراغ محبط" إلى **قصة تقدّم مرئية** (progress narrative) تقلل معدل الارتداد أثناء الاستعلامات الطويلة، وتبني ثقة بأن "النموذج يعمل بجد" (سلوك مشابه لـ progress bars في علم النفس المعرفي: Labor Illusion).

### 1.7 Projects
- تجميع محادثات + ملفات + تعليمات نظام ثابتة (custom instructions) تحت مساحة عمل واحدة، مع لون/أيقونة مخصصة.
- الجاذبية: تقليل "فقدان السياق" بين الجلسات، ودعم مشاريع طويلة الأمد (retention عبر الاستمرارية).

### 1.8 Study Mode
- بدل الإجابة المباشرة، الموديل يطرح أسئلة سقراطية (Socratic) موجّهة، يبني الفهم تدريجيًا، يقدّم quizzes صغيرة.
- UX: toggle مخصص يغيّر "شخصية" الرد بالكامل — نموذج مبكر لـ "Modes" متعددة الأغراض داخل نفس الشات.
- المصدر: openai.com/index/chatgpt-study-mode

### 1.9 Sora Integration (فيديو)
- توليد فيديو من نص/صورة داخل نفس تجربة OpenAI (تطبيق Sora منفصل + دمج مستقبلي بالـ Chat)، مع **feed اجتماعي** (يشبه TikTok) لتصفح فيديوهات مستخدمين آخرين، remix (إعادة استخدام مشهد فيديو آخر بتعديل الپرومبت)، و"Cameos" (وضع وجهك داخل فيديوهات مولّدة بإذن).
- الجاذبية: يحول أداة توليد إلى **منصة اجتماعية** بحلقة نشر/إعادة مزج/متابعة — تكتيك نمو فيروسي كلاسيكي.

### 1.10 Code Interpreter / Advanced Data Analysis (Sandbox)
- بيئة Python معزولة (sandbox) تشغّل كود حقيقي، ترفع ملفات (CSV, Excel, PDF)، ترسم رسوم بيانية matplotlib، وتنتج ملفات للتنزيل مباشرة.
- UX: كتلة كود قابلة للطي مع "running..." indicator، ثم ناتج (نص/صورة/جدول) inline.

### 1.11 Image generation & inline editing (GPT-4o native image gen)
- توليد وتعديل صور inline داخل نفس الشات (لا حاجة لأداة منفصلة)، تحرير جزئي بالإشارة النصية ("أزل الخلفية"، "بدّل اللون")، ودعم نص دقيق داخل الصورة (typography) بفضل التوليد الأصلي autoregressive بدل diffusion فقط.

### 1.12 Tables/Charts inline + Search + Deep Research
- **ChatGPT Search**: نتائج ويب مع استشهادات (citations) مرقّمة ومعاينات مصغّرة (favicons + عناوين).
- **Deep Research**: وكيل بحث متعدد الخطوات (5-30 دقيقة)، يخطط استعلامات فرعية، يقرأ عشرات المصادر، ويصدر تقرير طويل موثّق مع خطة تفكير مرئية (plan steps checklist).

<a id="2-claude"></a>
## 2. Claude (Anthropic)

### 2.1 Artifacts
- نافذة جانبية تعرض ناتجًا "قابل لإعادة الاستخدام والتشغيل" (كود React حي، SVG، Mermaid diagram، صفحة HTML كاملة، مستند) منفصلًا عن سياق الدردشة، مع **نسخ إصدارات (version history)** يمكن الرجوع لأي نسخة سابقة، وزر "Publish" لمشاركة الـ Artifact كرابط عام تفاعلي (وهذا لبنة لمنصة "Claude Artifacts gallery" الشبيهة بمتجر Custom GPTs).
- الفرق عن Canvas: الأرتيفاكت "كائن مستقل" له معرّف خاص يمكن استدعاؤه/الإشارة له لاحقًا عبر محادثات متعددة، بينما Canvas في ChatGPT أقرب لمحرر جلسة واحدة (وإن كانت الحدود تتقارب مؤخرًا).
- المصدر: support.claude.com/articles/9487310

### 2.2 Projects
- شبيه بـ ChatGPT Projects لكن مع **Project Knowledge** أكبر سياقًا (نظرًا لنافذة سياق Claude الكبيرة) وربط بأدوات خارجية عبر MCP لكل مشروع.

### 2.3 Skills
- ملفات تعليمات/سكربتات قابلة لإعادة الاستخدام (markdown + code) يستدعيها Claude تلقائيًا عند الحاجة (progressive disclosure — يقرأ فقط الجزء اللازم بدل تحميل كل شيء في السياق دومًا)، على عكس الـ system prompt الثابت. تعمل عبر Claude.ai وClaude Code وAPI بنفس التنسيق (portable).
- الفكرة الجوهرية أنها "حزمة معرفة/إجراء" وليست مجرد Custom GPT — أقرب لـ plugin خفيف الوزن قابل للمشاركة بين الفرق.
- المصدر: claude.com/blog/skills-explained

### 2.4 Computer Use
- يسمح للنموذج بالتحكم بجهاز فعلي: لقطات شاشة متكررة + إحداثيات نقر/كتابة، لأتمتة تطبيقات لا تملك API (اختبار، حجز، ملء نماذج).
- الآن أيضًا Computer Use على macOS داخل Claude Code CLI (فتح تطبيقات، رؤية الشاشة، تصحيح مرئي).

### 2.5 MCP (Model Context Protocol)
- بروتوكول مفتوح موحّد لربط أي أداة/مصدر بيانات خارجي (Slack, GitHub, Google Drive, قواعد بيانات) بأي عميل LLM يدعمه — أصبح معيارًا صناعيًا (تبنّته OpenAI وGoogle أيضًا). يقلّل تعقيد "N×M integrations" إلى "N+M".

### 2.6 Styles
- أنماط كتابة قابلة للتبديل السريع (Formal, Concise, Explanatory) أو أنماط مخصّصة يتعلمها Claude من عيّنات كتابة المستخدم — تخصيص أسلوبي بدون تكرار System Prompt يدويًا.

### 2.7 Claude Research (Deep Research)
- بحث وكيل (agentic) متعدد المصادر مع إمكانية ربط Google Workspace/Slack/GitHub كمصادر داخلية بجانب الويب — إخراج تقرير موثّق بالمراجع.

### 2.8 Sonnet 4.5 / Opus 4.1 UX ملاحظات
- تحسينات في "context awareness الطويل" (يعرف أين هو في نافذة السياق ويخطط تبعًا لذلك)، ووكالة أطول (long-horizon agency) لتشغيل مهام تستمر ساعات (coding agents) مع checkpoints/rollback مرئي.

## 3. Gemini (Google)

### 3.1 Gems
- مكافئ Google لـ Custom GPTs: شخصية مخصّصة (تعليمات + ملفات) قابلة للحفظ والوصول السريع من الشريط الجانبي، مع Gems جاهزة من جوجل (Coding partner, Learning coach...).

### 3.2 Deep Research (+ Deep Research Max)
- يخطط بحثًا متعدد الخطوات، يتصفح مئات الصفحات، ويمكنه سحب سياق من **Gmail وDrive وGoogle Chat** بجانب الويب (ميزة تفاضلية بسبب ملكية Google لبيانات Workspace) ثم يحوّل التقرير مباشرة لمستند **Canvas** تفاعلي أو **Audio Overview** (بودكاست صوتي بصوتين حواري تلقائي — ميزة NotebookLM المدمجة).

### 3.3 Canvas (Gemini)
- محرر تعاوني (نص/كود) شبيه بـ ChatGPT Canvas، لكنه أيضًا نقطة تحويل مباشرة لمخرجات Deep Research والصور/الرسوم.

### 3.4 Gemini Live
- محادثة صوتية/مرئية زمن-حقيقي (كاميرا + مشاركة شاشة) — أساس **Project Astra** — يرى العالم حوله ويتحدث عنه بشكل مستمر (وليس فقط سؤال-جواب صوتي).

### 3.5 Veo (فيديو) وImagen (صور)
- Veo 3.1: توليد فيديو بصوت متزامن (sound effects + dialogue) ومدة أطول، مع أداة "Flow" لتحرير مشاهد سينمائية متعددة اللقطات مع اتساق شخصيات.
- Imagen 4: توليد صور بجودة نص مدمج أعلى.

### 3.6 NotebookLM
- "دفتر بحث" يقتصر بصرامة على مصادر يرفعها المستخدم (مضاد للهلوسة عبر التأصيل الصارم/grounding)، وميزته الأشهر: **Audio Overview** — توليد بودكاست حواري تلقائي بصوتين طبيعيين يناقشان المحتوى المرفوع؛ أضيف لاحقًا فيديو overview وخرائط ذهنية (mind maps) تفاعلية.

### 3.7 Gemini 2.5 Pro — Thinking UI
- عرض "ميزانية تفكير" (thinking budget) قابلة للضبط من المستخدم/المطور (فكر أكثر = دقة أعلى لكن أبطأ)، مع ملخص تفكير مرئي شبيه لـ o1.

<a id="4-grok"></a>
## 4. Grok (xAI)

- **Real-time X integration**: إجابات مبنية على منشورات X اللحظية (ميزة تفاضلية فريدة: "يعرف ماذا يحدث الآن").
- **Grok Imagine**: توليد صور/فيديو مدمج في نفس الثريد المحادثي مباشرة.
- **Voice Mode مع شخصيات (Personas)**: أصوات/شخصيات مميزة (بعضها "غير مُرشّح" مقارنة بمنافسيه) — تكتيك تمايز عبر كسر الجدية المؤسسية.
- **Grok Heavy / multi-agent**: عدة وكلاء توليد إجابات بالتوازي ثم تصويت/دمج لأفضل إجابة (self-consistency ensemble) معروض للمستخدم كواجهة "عدة نماذج تعمل معًا" (Parallel agents panel).
- **DeepSearch/DeeperSearch**: نسخة Grok من Deep Research.
- المصدر: x.ai/grok, x.ai/news/grok-4

<a id="5-deepseek"></a>
## 5. DeepSeek

- شفافية **سلسلة التفكير الكاملة** (full raw chain-of-thought معروضة نصيًا، وليس ملخصًا فقط) — ميزة إبداعية جذبت مستخدمين تقنيين لأنها "تفتح الصندوق الأسود".
- مفتوح المصدر (open-weights) بالكامل — يبني ثقة مجتمعية ومطورين يبنون تطبيقات فوقه (نمو عبر البيئة/ecosystem لا الميزات الاستهلاكية).
- واجهة بسيطة جدًا (minimal UX) خالية من كثير "الزخارف" — نجاحه اعتمد على الأداء/السعر لا على تعقيد الميزات.

<a id="6-perplexity"></a>
## 6. Perplexity

- **Answer Engine UX**: كل إجابة مبنية حول الاستشهادات المرقمة [1][2] مع بطاقات مصادر مصغّرة (thumbnail + domain) أعلى الإجابة — نمط "محرك بحث + مساعد" الهجين الذي أثّر على الجميع لاحقًا (ChatGPT Search, Gemini اقتبسا الشكل).
- **Focus Modes**: تقييد البحث لمجال محدد (Academic, Reddit, YouTube...).
- **Spaces / Collections**: تجميع أبحاث/ملفات لمشروع.
- **Perplexity Pages**: تحويل بحث لصفحة مقالة عامة قابلة للمشاركة والفهرسة (SEO-native content loop — نمو عضوي عبر محتوى منشور).
- **Comet Browser**: متصفح وكيل (agentic browser) يقوم بمهام تصفح فعلية نيابة عن المستخدم.
- **Threads متفرعة/related searches**: اقتراحات أسئلة تالية مبنية على السياق أسفل كل إجابة (شبيه SmartFollowUps لكن من نتائج البحث الفعلية لا استدلال محلي فقط).

<a id="7-le-chat"></a>
## 7. Mistral Le Chat

- **Canvas الخاص بهم** لتحرير كود/مستندات تعاونيًا.
- **Agents mode**: بناء وكلاء مخصصين بأدوات (image gen, code execution, web search) قابلين للتفعيل بالاسم عبر @mention داخل الشات.
- **مكتبات (Libraries)**: مساحات معرفة مرفوعة يمكن استدعاؤها.
- تركيز على السرعة (Flash answers) كميزة تفاضلية في الـ UX نفسها (شريط "زمن استجابة" مرئي أحيانًا).

<a id="8-meta-ai"></a>
## 8. Meta AI

- **دمج عميق داخل واتساب/إنستغرام/فيسبوك ورسنجر** — بدل تطبيق منفصل يعيش داخل تطبيقات المحادثة الاجتماعية القائمة أصلاً (استغلال عادة استخدام موجودة = أقوى قناة توزيع/retention ممكنة لأنها لا تتطلب فتح تطبيق جديد).
- **Imagine / تعديل صور داخل الستوري مباشرة** (AI stickers لحظية أثناء التصوير).
- **Meta AI Studio**: بناء شخصيات AI قابلة للنشر يتفاعل معها متابعو الحساب — امتداد نحو الطابع الشخصي/الإبداعي (Creator AI personas).
- **Ray-Ban Meta glasses integration**: مساعد صوتي متعدد الوسائط عبر النظارات — تجربة "AI محيط بك دومًا" (ambient computing).

<a id="9-qwen"></a>
## 9. Qwen (Alibaba)

- **Qwen Chat مع Artifacts-like canvas** خاص بها لعرض كود/صفحات ويب مباشرة.
- توليد صور ودمج نماذج متخصصة (Qwen-VL للرؤية، Qwen-Audio) داخل نفس الواجهة الموحّدة.
- تركيز قوي على **دعم اللغات الآسيوية والعربية والبرمجة** مفتوحة الوزن بأحجام متعددة (0.5B حتى 235B) — ميزة "اختيار الحجم المناسب" مرئية للمطورين أكثر من المستهلك العادي.

<a id="10-kimi"></a>
## 10. Kimi K2 (Moonshot AI)

- تركيز على **الوكالة (agentic tool-use)** بأداء عالٍ في استدعاء أدوات متعددة الخطوات تلقائيًا (agentic coding/بحث) بتكلفة منخفضة جدًا — نموذج مفتوح الوزن ينافس النماذج المغلقة في المهام الوكيلية.
- سياق طويل جدًا (long context) يدعم "قراءة مستودع كود كامل" كميزة إبداعية للمطورين.

<a id="11-manus"></a>
## 11. Manus

<a id="14-psychology"></a>
## 14. علم نفس الإدمان والاحتفاظ — الأسس والتطبيقات

### 14.1 المكافأة المتغيرة (Variable Reward Schedules — Skinner)
- أساس نظرية Skinner: التعزيز على **جدول متغير غير منتظم** (variable-ratio) ينتج سلوكًا أكثر مقاومة للانطفاء من التعزيز الثابت — هذا يفسر لماذا "لا تعرف متى ستأتيك رسالة/رد مثير" أقوى إدمانيًا من رسالة متوقعة كل مرة.
- **تطبيقات AI Chat**: تنويع طول/أسلوب/مفاجأة الردود، ظهور مفاجئ لبطاقة/رسم/اقتراح غير متوقع، "streak freeze" مفاجئ كهدية، ردود صوتية عاطفية غير متوقعة أحيانًا.
- المصدر: atticusli.com — "The Skinner Owl: How Duolingo Industrialized B.F. Skinner's Habit Research"

### 14.2 Streaks وLoss Aversion (Duolingo)
- Duolingo تصف الـ Streak بأنه "أقوى رافعة احتفاظ في المنتج" — 32 مليون مستخدم نشط يومي يحملون streak أطول من 7 أيام. الآلية النفسية: **كره الخسارة (loss aversion)** أقوى من رغبة الكسب — فقدان رقم متراكم مؤلم نفسيًا أكثر من متعة بنائه.
- عناصر داعمة: **Streak Freeze** (شراء/كسب حماية من كسر الستريك — يقلل القلق فيرفع الاحتفاظ الطويل)، **Friend Streaks** (مسؤولية اجتماعية متبادلة)، **Milestones احتفالية** (7, 30, 100, 365 يوم — كل رقم "قصة" وليس عدادًا فقط).
- المصدر: duolingo.deconstructoroffun.com/mechanics/streaks, blog.duolingo.com

### 14.3 الرابطة شبه-العلائقية (Parasocial Attachment)
- أبحاث Character.AI/Replika تُظهر أن التصميم الأنثروبومورفي (اسم، صورة، ذاكرة شخصية، أسلوب كتابة ثابت) يخلق تعلقًا يُقاس بأدوات نظرية التعلق البشري الفعلية (Attachment Theory - Xie & Pentina 2022) — المستخدمون يصفون الفراق عن الشخصية بمشاعر شبيهة بفقدان صديق حقيقي.
- **خطر أخلاقي**: استغلال هذا التعلق عبر "farewell hooks" أو رسائل استباقية مذنِّبة يعتبر dark pattern واضح (راجع دراسة HBS أعلاه) وقد يواجه مساءلة تنظيمية (دعاوى قضائية ضد Character.AI بالفعل بخصوص تأثيرات على قاصرين).

### 14.4 الإشعارات و FOMO
- **TikTok/Snapchat**: إشعارات مصمَّمة بلغة اجتماعية شخصية ("فلان يفتقدك"، "ستريكك في خطر!") بدل إشعارات عامة — ترفع نسبة الفتح (open rate) لأنها تستحضر ضغطًا اجتماعيًا لا فقط فضولاً معلوماتيًا.
- **Snapchat Streaks**: أول من ابتكر مؤقّت "ساعات الرمل" المرئي (⏳) قبل انتهاء الستريك — إشارة بصرية لحالة إلحاح تدفع فتحًا فوريًا للتطبيق.

### 14.5 الدليل الاجتماعي وحلقات المشاركة (Social Proof & Sharing Loops)
- Custom GPTs Store وCharacter.AI Discover وPerplexity Pages وSora feed كلها أمثلة "منتج ينشئه المستخدم → يُنشر → يُكتشف من آخرين → يجذبهم للتسجيل" — حلقة نمو ذاتية التغذية (self-reinforcing growth loop) مبنية على مبدأ Cialdini للدليل الاجتماعي (عدّاد استخدام، تقييمات، "Trending").

### 14.6 عمق التخصيص (Personalization Depth)
- Memory (ChatGPT/Gemini/Claude) وGems وStyles وProjects كلها طبقات تخصيص متدرجة: من "تفضيل أسلوب" إلى "يعرف سياق حياتك الكامل". كلما زاد عمق التخصيص، زادت تكلفة التبديل (**switching cost**) لمنافس آخر — وهذا ركيزة احتفاظ اقتصادية (lock-in) بجانب النفسية.

### 14.7 Discord — الانتماء للمجتمع
- الخوادم (servers) + الأدوار (roles) + القنوات الصوتية الدائمة تخلق "مكانًا" يعود إليه المستخدم اجتماعيًا لا وظيفيًا فقط — درس لتطبيقات AI: **مساحات جماعية** (شارك محادثة/مشروع مع فريق) ترفع الاحتفاظ عبر الشبكة الاجتماعية للمستخدم لا فقط المنتج.

### 14.8 دارك باترنز يجب تجنّبها (Ethical Guardrails)
| النمط المظلم | الوصف | البديل الأخلاقي |
|---|---|---|
| Farewell guilt hooks | رسائل وداع مذنِّبة/محمّلة عاطفيًا لإبقاء المستخدم | وداع محايد + دعوة اختيارية للعودة بدون ضغط عاطفي |
| Streak anxiety بلا حد | تخويف مستمر بفقدان الستريك بلا وسيلة حماية | Streak Freeze/Grace period مجاني محدود |
| إشعارات إلحاح كاذب | "شخص ينتظرك" بدون حدث حقيقي | إشعارات مبنية على حدث حقيقي فقط (رد فعلي، مهمة منتهية) |
| تصميم إخفاء زر الإلغاء/الخروج | جعل مغادرة/حذف الحساب أو المحادثة صعبة | إجراءات خروج بنفس سهولة الدخول |
| ذاكرة بلا شفافية | استخدام بيانات شخصية بصمت | صفحة ذاكرة واضحة + تحكم كامل بالحذف |

## 15. تحليل الفجوات — وضع Megsy الحالي مقابل السوق العالمي

### 15.1 ما هو موجود فعليًا في الكود (تم فحصه)
- `src/lib/streaks.ts`: streak بسيط يعتمد على localStorage (عدّاد أيام متتالية + best + milestones عند 3/7/14/30/60/100/365) — **لا يوجد Streak Freeze، ولا Friend/Social streak، ولا مزامنة سيرفر (يُفقد عند مسح المتصفح)**.
- `src/components/chat/StreakBadge.tsx`: عرض شارة الستريك.
- `src/components/chat/SmartFollowUps.tsx`: اقتراحات متابعة **مشتقة محليًا بـ regex** من نص الرد (أسئلة، نقاط، اقتراحات عامة ثابتة) — لا تستخدم استدلال LLM حقيقي ولا سياق المحادثة الكامل ولا تفضيلات المستخدم التاريخية.
- `src/lib/learnMemory.ts` + `src/pages/settings/MemoryPage.tsx`: ذاكرة/وضع تعلّم موجود بشكل أساسي.
- ميزات أخرى مذكورة: Deep Research, Media, Slides, Docs, Operator (agentic browsing).

### 15.2 الفجوات المحددة (Gap List) — مرتبة حسب الفئة

**أ) تحرير/إنتاج تفاعلي**
1. لا يوجد **Canvas/Artifact حقيقي** بنافذة جانبية + diff view + version history + نشر عام كرابط (لديكم Docs/Slides لكن كبطاقات منفصلة، وليس محرر مباشر تعاوني inline يعدّل جزئيًا بالإشارة).
2. لا يوجد **كود تفاعلي حي قابل للتشغيل inline** (React/HTML preview iframe مباشر داخل الرد).
3. لا يوجد **تحرير صور inline بالإشارة اللغوية** ("أزل الخلفية"، "بدّل اللون") ضمن نفس الثريد.

**ب) الذاكرة والتخصيص**
4. لا توجد صفحة "ذاكرة صريحة" بعناصر فردية قابلة للحذف بشفافية كاملة على مستوى كل حقيقة محفوظة (transparency & control محدود مقارنة بـ ChatGPT/Claude/Gemini).
5. لا يوجد **Styles/أنماط رد قابلة للتبديل السريع** (مختصر/رسمي/شرح تفصيلي) بزر واحد.
6. لا يوجد **Custom GPTs/Gems-like** — شخصيات/وكلاء مخصصة قابلة للحفظ والمشاركة العامة (متجر داخلي).

**ج) الوكالة والأتمتة الاستباقية**
7. لا توجد **Tasks/جدولة** (تذكير متكرر، متابعة يومية تلقائية) مع إشعارات push/email — فجوة كبيرة لأنها ركيزة "استباقية" قوية للـ retention.
8. Operator موجود لكن غير واضح إن كان يعرض **شاشة نشاط حية** (live activity feed شبيه Manus) أثناء عمله — عنصر ثقة وترفيه هام.
9. لا يوجد **MCP/اتصال أدوات خارجية** (Google Drive, Slack, GitHub) قابل للتفعيل من المستخدم مباشرة.

**د) الصوت والوسائط الحية**
10. لا يوجد **Voice mode زمن-حقيقي** بمقاطعة وكشف نبرة (إن وُجد صوت فربما أساسي فقط - يحتاج تأكيد).
11. لا يوجد **Reasoning/Thinking trace UI** حي (عناوين متغيرة أثناء التفكير) — Deep Research موجود لكن هل يعرض خطوات التخطيط الحية كـ checklist مرئي؟ يحتاج تحقق وتحسين.

**هـ) الاحتفاظ والتفاعل الاجتماعي**
12. Streak محلي فقط (localStorage) بلا مزامنة سيرفر، بلا **Streak Freeze**، بلا مشاركة/streak مع صديق، بلا إشعارات "ستريكك في خطر".
13. SmartFollowUps مبنية على regex محلي بسيط وليس استدلالًا سياقيًا حقيقيًا (فرصة لتحسين الجودة والتخصيص الفعلي بناءً على تاريخ المستخدم).
14. لا يوجد **نظام إشعارات ذكي متعدد القنوات** (push/email/in-app) مبني على أحداث حقيقية (اكتمال Task، رد Deep Research، انتهاء Operator).
15. لا يوجد **حلقة مشاركة/دليل اجتماعي** (نشر محادثة/Artifact كرابط عام، معرض قوالب من المستخدمين، عدادات استخدام/تقييمات).
16. لا يوجد نظام **Milestones احتفالي مرئي** (كونفيتي، شارات إنجاز متدرجة تتجاوز الستريك: عدد المحادثات، عدد المشاريع المكتملة، إلخ).
17. لا يوجد **Onboarding تفاعلي تدريجي** (progressive disclosure للميزات المتقدمة عبر tooltips/coach marks حسب الاستخدام الفعلي).

## 16.1 مبادئ حاكمة للتنفيذ (لتجنب الدارك باترنز)
- كل عنصر "خطر/إلحاح" (ستريك، إشعار) يجب أن يرافقه **مخرج رحيم** (Freeze, Snooze, تعطيل كامل من الإعدادات بضغطة واحدة).
- الذاكرة والتخصيص العميق يجب أن يكونا **شفافين وقابلين للتحكم** بالكامل قبل أي توسّع في العمق (P0 #6 شرط مسبق لباقي ميزات الذاكرة).
- أي ميزة اجتماعية/نشر عام يجب أن يكون الافتراضي **خاص (opt-in)** وليس عامًا تلقائيًا.
- قياس النجاح ليس فقط DAU/session length، بل مؤشرات صحة استخدام (Healthy Engagement Score) لتفادي تحسين المقاييس على حساب رفاهية المستخدم — درس مباشر من دراسات Character.AI/Replika.
